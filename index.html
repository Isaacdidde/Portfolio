LetsDefend · EventID 118 Command Injection True Positive HIGH

SOC168 — Whoami Command Detected in HTTP Request Body

One of the most frequently misclassified alerts on LetsDefend. The attacker injected OS commands via POST parameter and escalated from whoami to cat /etc/shadow within minutes. HTTP 200 responses with file-sized payloads and EDR terminal history confirmed active RCE and credential exposure on WebServer1004.

T1059T1082T1552SplunkVirusTotal
★ Commissioned by LetsDefend for community publication
Read on Medium ↗

// 01 · Alert Received

EventID 118 triggered — whoami string detected in HTTP POST request body to WebServer1004. Alert classified as Medium by the SIEM; triage initiated to validate severity.

// 02 · Initial Triage

Checked source IP via VirusTotal and threat intel lookups — flagged as known malicious. HTTP response was 200 OK. Payload size in response body consistent with successful command execution output.

// 03 · Investigation

Correlated in Splunk: source IP had no prior legitimate traffic to this endpoint. EDR terminal history on WebServer1004 showed command chain: whoami → id → uname -a → cat /etc/shadow. All returned HTTP 200 with non-empty bodies.

// 04 · IOC Analysis

Source IP: External, flagged malicious. Endpoint: POST /vulnerable-param. Payload: OS command injection via unsanitized parameter. Evidence: HTTP 200 with payload-sized response body, EDR terminal history, Splunk log correlation.

// 05 · Verdict & Escalation

True Positive — Critical. Active RCE confirmed. Credential file (/etc/shadow) read by attacker. Host isolated; escalated to Tier 2 with full IOC package and timeline documentation.

// 06 · Lessons Learned

HTTP response size and code together are better indicators than the alert string alone. Always correlate EDR terminal history with SIEM logs before downgrading severity. This case is commonly false-negatived due to misconfigured alert thresholds.
